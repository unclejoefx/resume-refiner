# Resume Refiner - Project Plan

## Project Overview

Resume Refiner is a web application that helps users improve their resumes through AI-powered analysis, ATS optimization, grammar checking, and format standardization.

## Technology Stack

### Backend
- **Framework**: Python with FastAPI
- **AI Integration**: Anthropic Claude API
- **Document Processing**:
  - PyPDF2 or pdfplumber (PDF parsing)
  - python-docx (Word document handling)
  - reportlab or WeasyPrint (PDF generation)
- **Grammar Checking**: LanguageTool or grammar-check library
- **Storage**:
  - File system for temporary document storage
  - PostgreSQL or SQLite for user data and analysis history
- **Environment Management**: python-dotenv for configuration

### Frontend
- **Framework**: React with TypeScript
- **Styling**: Tailwind CSS or Material-UI
- **State Management**: React Context API or Redux Toolkit
- **File Upload**: react-dropzone
- **Document Preview**: react-pdf or PDF.js
- **HTTP Client**: Axios

### Development Tools
- **Backend**: pytest, black, flake8, mypy
- **Frontend**: Vite, ESLint, Prettier
- **Version Control**: Git
- **API Documentation**: OpenAPI/Swagger (auto-generated by FastAPI)

## Core Features

### 1. Document Upload and Parsing
- Support for PDF and DOCX formats
- Extract text content and structure
- Preserve formatting information for analysis

### 2. AI-Powered Content Improvement
- Use Claude API to analyze resume content
- Suggest improvements for:
  - Bullet points and achievements
  - Professional summary
  - Skills presentation
  - Action verbs and impact statements
- Provide explanations for each suggestion

### 3. ATS (Applicant Tracking System) Optimization
- Keyword analysis based on job descriptions
- Format compatibility checking
- Section header standardization
- Score resume for ATS-friendliness
- Suggestions for keyword optimization

### 4. Grammar and Spell Checking
- Integrate grammar checking library
- Highlight errors with suggestions
- Check for:
  - Spelling errors
  - Grammar mistakes
  - Punctuation issues
  - Consistency in tense and voice

### 5. Format Standardization
- Convert between PDF and DOCX
- Apply consistent formatting templates
- Ensure proper structure:
  - Contact information
  - Professional summary
  - Work experience
  - Education
  - Skills
  - Optional sections (certifications, projects, etc.)

### 6. Analysis Dashboard
- Overall resume score
- Section-by-section breakdown
- Visual indicators for improvements
- Download refined resume

## API Endpoints

### Upload
- `POST /api/upload` - Upload resume file
- `GET /api/upload/{upload_id}` - Get upload status

### Analysis
- `POST /api/analyze` - Analyze uploaded resume
- `GET /api/analyze/{analysis_id}` - Get analysis results
- `POST /api/analyze/grammar` - Run grammar check
- `POST /api/analyze/ats` - Run ATS optimization
- `POST /api/analyze/content` - Run AI content analysis

### Export
- `GET /api/export/{analysis_id}/pdf` - Download as PDF
- `GET /api/export/{analysis_id}/docx` - Download as DOCX

## Security Considerations

- Validate file types and sizes on upload
- Sanitize file content before processing
- Use secure file storage with unique identifiers
- Implement rate limiting on API endpoints
- Store Claude API key securely (never in code)
- Clean up temporary files after processing
- Implement CORS properly
- Add authentication for production (optional for MVP)
